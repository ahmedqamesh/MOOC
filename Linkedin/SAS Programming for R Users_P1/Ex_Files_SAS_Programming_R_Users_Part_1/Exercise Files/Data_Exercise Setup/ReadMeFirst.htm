Setting Up Data for This Course

These instructions are primarily for the download version of SAS University Edition, with notes for users accessing the AWS version.

IMPORTANT: You must perform tasks 2 and 3 in the same SAS Studio session. Don't close your browser until you've completed both tasks or you will need to start again.

Note:
Some demonstrations in Course Four: The Interactive Matrix Language and Open Source Integration show examples of integrating R with SAS. Integration with R requires an installed version of R, plus a few additional configuration settings within your SAS environment. Because you cannot modify its configuration, SAS University Edition does not support integration with R. For this course, you can learn by watching the demonstrations.

Task 1: Download the course files.

  1. Open the SP4R_Data.zip file. This file contains SAS programs and text files for this course.

  2. On your computer, locate and open the SASUniversityEdition folder, and then open the myfolders folder. Create a new folder named sp4r.
    Note: If you don't have a myfolders folder, go back to the SAS University Edition Quick Start Guide and follow the instructions to set it up before you continue.

  3. Extract the contents of the zip file to the sp4r folder.

For SAS University Edition on AWS:

  1. Open the SP4R_Data.zip file. and extract its contents to a local folder. This file contains SAS programs and text files for this course.
  2. Log into SAS University Edition on AWS.
  3. In the Server Files and Folders pane, click New and select Folder. Create a folder named sp4r.
  4. Expand My Folders and select sp4r.
  5. Click Upload and then click Choose Files. Navigate to the location where you stored the contents of the zip file, select all the files, and upload them to the sp4r folder.

Task 2: Define the sp4r library.

  1. Start SAS Studio from your SAS University Edition: Information Center.

  2. In the Server Files and Folders pane, click New and select SAS Program.

  3. The following code creates a macro variable to store the location of your course data and defines the sp4r library. Copy and paste the following two lines of code into the Code tab in SAS Studio.

    %let path=/folders/myfolders/sp4r; 
libname sp4r "&path";
  4. Click Run to submit this program to SAS.

  5. Check the log to make sure that the libref sp4r was successfully assigned. You should see this message in the log:

    NOTE: Libref SP4R was successfully assigned

    Show troubleshooting tips

    If the program does not run successfully, check the path to the sp4r folder:
    • In the Server Files and Folders pane, open My Folders.
    • Right-click sp4r and select Properties. Verify that the location is /folders/myfolders/sp4r.
    • If the path is different, copy the path shown in Location and replace the filepath shown in line 1 of the code.
    • Run the code again.

    If the program still does not run, go back to the SAS University Edition setup instructions and make sure you created the myfolders subfolder and defined it as a shared folder in your virtualization software.



  6. Click the Code tab and click Save . In the Save As window, open the sp4r folder. Name the program setup.sas and click Save.

    Optional: If you prefer, you can add this code to the Snippets panel. To do this, click Add to My Snippets and type setup in the Name field. To run the program, press F4 to open a new program tab, open the Snippets panel, open My Snippets, and double-click setup. Then, run the code.

  7. Close the setup.sas program tab in the workspace.

IMPORTANT: Each time you start SAS Studio, you must open the setup.sas program and run it to define the sp4r library so that you have access to your course data.

Task 3: Create the SAS data sets for the course — you only have to do this once.

  1. In the Server Files and Folders pane, open My Folders and then open the spr4 folder.

  2. Double-click create_data.sas to open it.

  3. Click Run to submit the program to SAS. This program creates SAS data sets in the sp4r folder.

  4. After the code runs, the Results tab displays two tables. The second table lists the data sets that the program created. Check the log to verify that there are no errors or warnings. There will be many notes because SAS writes a note for each data set that the program creates.

    NOTE: If your program does not run successfully, make sure you completed all the steps in Task 2 correctly.

  5. Close the create_data.sas program tab in the workspace.

    NOTE: Unless you delete your sp4r folder, you do not need to perform this task again.

For SAS University Edition on AWS:

  • When you stop an instance of SAS University Edition (or if the instance times out), the course data persists. When you start the instance again, you need to open and submit the setup.sas program to define the sp4r library.
  • When you terminate an instance of SAS University Edition, the course data is deleted along with the instance. You must perform Task 2 and Task 3 when you create a new instance.
